  \item Образец использования: невозможно передать параметр.
  Более подробно: стандартная функция обработчика сигнала имеет прототип
\begin{verbatim}
  void fun(int signal)
\end{verbatim}
  На практике нам надо передавать туда параметры. Обычно это делается с 
  помощью глобальных переменных.
  Следовательно, нам надо самим определить  область глобальных переменных.
  Может быть имеет смысл определить обработчик события как пользовательский 
  класс:
\begin{verbatim}
class SignalHandler
{
public:
  virtual ~SignalHandler();
  virtual void run(int signal) = 0;
};
\end{verbatim}
  \begin{itemize}
    \item Действия: продумать образец использования и встроить в Signalling
     эту функциональность>
    \item Сроки:
    \item Ответственность:
  \end{itemize}
  \item Образец использования: невозможно определить одну функцию для
   совокупности сигналов.
   Так, например, если нам нужно поставить функции на все сигналы, то
   необходимо что-то вроде:
   \begin{verbatim} 
     Signalling(fun,0); 
     Signalling(fun,2); 
     ..................
     Signalling(fun,15); 
#if defined(NSIG) && NSIG=32
     Signalling(fun,16); 
     ....
     Signalling(fun,31); 
#endif
   \end{verbatim} 
    Понятно, что такой код можно только в страшном сне написать.
   \begin{itemize}
    \item Действия: Определить маску сигналов как битовый флаг, с константами,
    имеющие те-же имена, чтол и в станадрате с префиксом en.
    Определить функции: haveSignal(int) которая возворащает true или false,
    в зависимости от того: есть ли такой сигнал в данной ОС.
    Также определить Signalling::enALL
    \item Сроки:
    \item Ответственность:
